## 一、补卡审批流程

│  │ • 验证当月补卡次数是否已达上限（≤3次）                  │   │

│ - E3005 超出补卡期限   │   │ 同时发送消息通知部门管理员        │
│ - E3002 非研发行政员工  │   └─────────────┬─────────────────┘
└─────────────────────┘                     │
                                            │ 2. 通知部门管理员
                                            ▼
                          ┌─────────────────────────────────┐
                          │         部门管理员                │
                          │  GET /api/approval/pending      │
                          │  type="补卡"                     │
                          └─────────────┬───────────────────┘
                                        │
                    ┌───────────────────┼───────────────────┐
                    │                   │                   │
                    ▼                   ▼                   ▼
            ┌───────────────┐   ┌───────────────┐   ┌───────────────┐
            │   查看详情     │   │   通过        │   │   驳回        │
            │ 补卡日期/原因  │   │   填写意见    │   │   填写意见    │
            └───────┬───────┘   └───────┬───────┘   └───────┬───────┘
                    │                   │                   │
                    └───────────────────┼───────────────────┘
                                        │
                    ┌───────────────────┴───────────────────┐
                    │                                       │
                    ▼                                       ▼
        ┌─────────────────────┐               ┌─────────────────────┐

        │ /补卡/:id            │               │ /补卡/:id           │

### 补卡状态机

```
[提交申请] ──→ [待审批] ──→ [已通过]  (打卡记录更新)
                  │
                  └──→ [已驳回]  (流程结束)
```

## 三、销售外出打卡流程（简化）

```
┌─────────────────────────────────────────────────────────────┐
│                    销售线员工                                │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          │ 1. 打开小程序首页
                          │    显示：今日打卡状态（未打卡/已打卡）
                          │
                          │ 2. 点击"外出打卡"
                          │    POST /api/attendance/checkin
                          │    - latitude/longitude: GPS坐标
                          │    - location_name: 位置描述
                          │    - photo_base64: 拍照（可选）
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐

│  ┌─────────────────────────────────────────────────────┐   │
│  │ • 验证员工类型为销售线                                 │   │
│  │ • 验证今日打卡次数是否已达上限（默认1次）                │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────┬───────────────────────────────────┘
                          │
              ┌───────────┴───────────┐
              │                       │

              ▼                       ▼
┌─────────────────────┐   ┌─────────────────────────────────┐
│ 返回错误:             │   │ 创建打卡记录                     │

└─────────────────────┘   └─────────────┬───────────────────┘
                                        │
                                        ▼
                        ┌─────────────────────────────┐
                        │ 返回打卡结果                   │
                        │ { record_id, status,        │
                        │   check_in_time, message }   │
                        └─────────────────────────────┘
```

## 四、研发行政固定打卡流程

```
┌─────────────────────────────────────────────────────────────┐
│                    研发行政员工                              │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          │ 1. 到达公司范围内，打开小程序
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      打卡规则判断                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 读取考勤规则: attendance_rule                        │   │
│  │ • checkin_times: 07:00-09:00 / 17:00-23:59         │   │
│  │ • location: {lat, lng, radius: 500m}               │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          │ 2. 点击"打卡"
                          │    POST /api/attendance/checkin
                          │
                          ▼
        ┌─────────────────────────────────────────┐
        │ 系统校验:                                 │
        │ ① GPS位置是否在允许范围内（500m误差）       │
        │ ② 当前时间是否在打卡时段内                 │
        └─────────────────┬───────────────────────┘
                          │
              ┌───────────┼───────────┐
              │           │           │
              ▼           ▼           ▼
      ┌───────────┐ ┌───────────┐ ┌───────────┐
      │ 位置超出   │ │ 未到时间   │ │ 校验通过  │
      │ 返回错误   │ │ 返回错误   │ │           │
      │ E3001     │ │ E3002     │ │           │
      └───────────┘ └───────────┘ └─────┬─────┘
                                        │
                                        ▼
                        ┌─────────────────────────────┐
                        │ 判定打卡状态:                 │
                        │ • 07:00-09:00 → 正常/迟到    │
                        │ • 17:00-23:59 → 正常/早退    │
                        └─────────────┬───────────────┘
                                      │
                                      ▼
                        ┌─────────────────────────────┐
                        │ 创建打卡记录                 │
                        │ status: 正常/迟到/早退        │
                        │ 返回打卡结果给小程序          │
                        └─────────────────────────────┘
```

## 五、异常考勤判定流程（每日定时任务）

```
┌─────────────────────────────────────────────────────────────┐
│                    系统定时任务（每日凌晨）                    │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          │ 扫描昨日所有研发行政员工
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                    考勤异常判定                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 对每个员工:                                           │   │
│  │                                                     │   │
│  │ 1. 查询昨日打卡记录数量                               │   │
│  │                                                     │   │
│  │ 2. 如打卡次数 < 2次:                                 │   │
│  │    → 缺勤 (Absent)                                   │   │
│  │                                                     │   │
│  │ 3. 如有请假记录覆盖昨日:                             │   │
│  │    → 不计入缺勤（视为请假）                          │   │
│  │                                                     │   │
│  │ 4. 如无请假 + 无打卡:                                │   │
│  │    → 旷工 (Absenteeism)                             │   │
│  │                                                     │   │
│  │ 5. 更新打卡记录状态                                   │   │
│  │    • 迟到: late_count +1, late_minutes              │   │
│  │    • 早退: early_leave_count +1, early_minutes      │   │
│  │    • 缺勤: absent_count +1                          │   │
│  │    • 旷工: absenteeism_count +1                     │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          │ 生成异常通知
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      消息通知                                │
│  • 异常提醒 → 发送给员工                                     │
│  • 部门管理员汇总 → 发送给部门管理员                           │
└─────────────────────────────────────────────────────────────┘
```

## 审批优先级规则

| 审批类型 | 默认审批人 | 审批时效 | 超时处理 |
|---------|----------|---------|---------|
| 补卡 | 部门管理员 | 24小时 | 提醒+升级 |
| 请假 | 部门管理员 | 24小时 | 提醒+升级 |

## 消息通知触发点

| 场景 | 接收人 | 通知类型 |
|------|-------|---------|
| 补卡申请提交 | 部门管理员 | 审批通知 |
| 请假申请提交 | 部门管理员 | 审批通知 |
| 补卡审批通过 | 申请人 | 审批结果 |
| 补卡审批驳回 | 申请人 | 审批结果 |
| 请假审批通过 | 申请人 | 审批结果 |
| 请假审批驳回 | 申请人 | 审批结果 |
| 打卡异常（迟到/缺勤） | 员工 | 异常提醒 |
| 补卡次数将满 | 员工 | 提醒通知 |
